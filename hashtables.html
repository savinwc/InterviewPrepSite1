Understanding Maps

Maps are ideal for key-value association mapping, similar to dictionaries. They are used to perform lookups by keys or when there is a need to retrieve and update elements using keys. Maps are versatile data structures that can efficiently manage relationships between unique keys and corresponding values.

Common Scenarios for Using Maps

  • Map of Error Codes and Their Descriptions: A map can be used to associate error codes with their descriptions. For instance: {"404": "Not Found", "500": "Internal Server Error", "403": "Forbidden"}
  • Map of Zip Codes and Cities: A map can store zip codes as keys and their corresponding cities as values. For example: {"10001": "New York", "90001": "Los Angeles", "60601": "Chicago"}
  • Map of Managers and Employees: Each manager (key) can be associated with a list of employees (value) they manage. For instance: {"John Smith": ["Alice", "Bob"], "Jane Doe": ["Charlie", "David"]}
  • Map of Classes and Students: Each class (key) can be mapped to a list of students (value) enrolled in that class. For example: {"Math": ["John", "Jane"], "Science": ["Alice", "Bob"]}

Map Methods

Method Action Performed
clear() Clears and removes all elements or mappings from a specified Map collection.
containsKey(Object key) Checks whether a particular key is mapped in the Map. Returns true if the key is present, otherwise false.
containsValue(Object value) Checks whether a particular value is mapped by any key in the Map. Returns true if the value is present, otherwise false.
entrySet() Creates a set view of the map's entries (key-value pairs). Returns a set of map entries.
equals(Object obj) Checks for equality between two maps. Returns true if the maps contain the same key-value pairs.
get(Object key) Retrieves or fetches the value mapped by the specified key. Returns null if no mapping exists for the key.
hashCode() Generates a hash code for the map based on its key-value pairs.
isEmpty() Checks if the map is empty (i.e., no key-value pairs exist). Returns true if the map is empty, otherwise false.
keySet() Returns a set view of the keys contained in the map. The set is backed by the map.
put(Object key, Object value) Associates the specified value with the specified key in the map.
putAll(Map m) Copies all mappings from the specified map to this map.
remove(Object key) Removes the mapping for the specified key if it is present in the map.
size() Returns the number of key-value pairs in the map.
values() Creates a collection view of the values contained in the map.
getOrDefault(Object key, V defaultValue) Returns the value to which the specified key is mapped, or defaultValue if no mapping exists for the key.
merge(K key, V value, BiFunction remappingFunction) Associates the specified key with the given value if it is not already associated, or updates it using the remapping function if it is.
putIfAbsent(K key, V value) Associates the specified key with the given value if the key is not already associated with a value. Returns null if the association was made; otherwise, returns the current value.

Code Example


import java.util.*;

// Main class
public class Main {

    // Main driver method
    public static void main(String[] args)
    {

        // Creating an empty HashMap
        Map<String, Integer> map = new HashMap<>();

        // Inserting entries in the Map
        // using put() method
        map.put("vishal", 10);
        map.put("sachin", 30);
        map.put("vaibhav", 20);

        // Iterating over Map
        for (Map.Entry<String, Integer> e : map.entrySet())

            // Printing key-value pairs
            System.out.println(e.getKey() + " " + e.getValue());
    }
}
Output:
vaibhav 20
vishal 10
sachin 30
